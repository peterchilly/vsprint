## 模型描述

ERes2NetV2 是一种基于 Res2Net 的增强型说话人识别模型，通过以下创新提升性能：

1. **分组特征提取**：多尺度层级化特征处理，捕获不同粒度的说话人特征
2. **特征复用机制**：后续分组复用前面分组的输出，增强信息流动
3. **通道交互模块**：轻量级 SE-Block 注意力机制，自适应校准通道权重
4. **选择性残差融合**：可学习权重的残差连接，自动平衡残差与映射
5. **注意力统计池化 (ASP)**：说话人识别标准池化方式，加权聚合时序信息

### 模型结构

```
输入 (1, 1, 80, 200)
  ├─ Stem: Conv2d(1→64) + BN + ReLU + MaxPool
  ├─ Stage 1: 3 × ERes2NetV2BasicBlock(64→64)
  ├─ Stage 2: 4 × ERes2NetV2BasicBlock(64→128)
  ├─ Stage 3: 6 × ERes2NetV2BasicBlock(128→256)
  ├─ Stage 4: 3 × ERes2NetV2BasicBlock(256→512)
  ├─ 频率均值池化 → AttentiveStatsPool
  ├─ BN + Dropout + FC(1024→192) → Embedding
  └─ Classifier: Linear(192→N) → Logits
```

## 预期用途

### 适用场景

- 说话人验证：判断两段音频是否来自同一说话人
- 说话人识别：从已注册的说话人中识别身份
- 说话人 embedding 提取：用于下游任务

### 不适用场景

- 跨语言说话人识别（模型在多语言数据上未充分验证）
- 远场/噪声环境下的识别（训练数据以近场为主）
- 实时流式说话人切换（模型设计为非流式）
- 语音合成/深度伪造检测（不具备防伪能力）

## 局限性

1. **数据规模**：仅在 VoxCeleb1 dev set 上训练，数据量有限
2. **说话人数量**：训练说话人约 1200 人，大规模场景需重新训练
3. **音频长度**：固定 200 帧（~2 秒），过短音频可能影响精度
4. **噪声鲁棒性**：未进行充分的噪声鲁棒性训练
5. **多语言泛化**：主要在英语数据上训练，多语言泛化性未验证

## 伦理考量

### 隐私保护

- 说话人 biometric 数据属于敏感个人信息，使用时应遵守当地法律法规
- 建议对存储的 embedding 进行加密
- 提供用户数据删除机制（API 支持 DELETE 操作）

### 公平性

- VoxCeleb 数据集可能存在人口统计学偏差
- 模型在不同年龄、性别、口音群体上的表现可能不均匀
- 部署前应针对目标群体进行公平性评估

### 滥用风险

- 说话人识别技术可能被用于未经授权的监控
- 建议仅在用户知情并同意的场景下使用
- 不应用于司法取证等高风险场景

### 深度伪造

- 本模型不具备检测语音合成/深度伪造的能力
- 在深度伪造攻击场景下，模型可能被欺骗
- 建议配合反欺骗模块使用
