# ERes2NetV2 模型训练与运行 Pipeline

> 基于 ERes2NetV2 模型架构，规划从零到部署的完整深度学习训练和运行 pipeline。

---

## 项目背景

本项目旨在实现 **ERes2NetV2**（Enhanced Res2Net V2）深度学习模型从环境搭建、数据准备、模型构建、训练、评估到部署的完整 pipeline。

ERes2NetV2 是一种改进的 Res2Net 架构，核心创新点包括：
- **分组特征提取**：多尺度层级化特征处理
- **特征重用机制**：后续分组复用前面分组输出，避免重复计算
- **通道交互模块**：轻量级注意力机制实现特征重校准
- **选择性残差融合**：可学习权重的残差连接

本项目面向图像分类 / 目标检测 / 语义分割等计算机视觉任务。

**详细架构解读**：`ERes2NetV2_深度解读.md`

## 工作流程

本项目采用 **Task → Execute → Verify** 的闭环工作流。每个任务的执行和验收都通过调用 Claude（Claude Code）完成。

### 工作流概览

```
开始
  │
  ▼
选择任务文档（tasks/ 下的某个 .md）
  │
  ▼
调用 Claude 执行任务
  │  使用: claude -p "<prompt 包含任务文档内容 + 执行指令>"
  │
  ▼
任务完成（代码/配置/文档产出）
  │
  ▼
调用 Claude 生成验证方案
  │  使用: python scripts/generate_verify.py <任务文档路径>
  │  输出: audit/<阶段>/<任务名>_verify.md
  │
  ▼
调用 Claude 执行验证
  │  使用: claude -p "<prompt 包含验证文档内容 + 验收指令>"
  │
  ▼
验证通过？
  ├─ 是 → 标记任务完成 ✅
  └─ 否 → 返回修改，重新执行
```

### 详细步骤

#### 第一步：选择任务

进入 `docs/superpowers/tasks/` 目录，选择要执行的任务文档。

每个任务文档包含：
- 所属阶段和阶段目标
- 具体的任务清单（带 checkbox）
- 验收标准

#### 第二步：调用 Claude 执行任务

使用 Claude Code 的 `-p`（print）模式执行任务：

```bash
# 在项目根目录执行
cd C:\Users\Administrator\vsprint

# 调用 Claude 执行指定任务
claude -p "请阅读以下任务文档，并完成该任务的所有工作：

$(cat 'docs/superpowers/tasks/01_环境搭建/01_硬件评估与选择.md')

请按照任务清单逐项执行，产出相应的代码、配置或文档。
将成果物保存到项目的正确位置。"
```

#### 第三步：生成验证方案

任务执行完成后，调用脚本自动生成验证文档：

```bash
python scripts/generate_verify.py tasks/01_环境搭建/01_硬件评估与选择.md
```

脚本会自动：
1. 读取任务文档内容
2. 调用 `claude -p` 应用 superpowers verification skill
3. 生成验证测试设计文档
4. 保存到 `audit/` 目录，命名为 `原任务名_verify.md`

#### 第四步：调用 Claude 执行验证

读取生成的验证文档，调用 Claude 执行验收：

```bash
claude -p "请阅读以下验证文档，并对上一步任务执行的产出进行验收：

$(cat 'docs/superpowers/audit/01_环境搭建/01_硬件评估与选择_verify.md')

请按照验证方案逐项检查任务执行结果，确认每个验收标准是否达标。
如果发现问题，请指出具体问题和建议的修改方案。"
```

#### 第五步：确认或修改

- **验证通过** → 在任务文档中勾选所有 checkbox，标记完成 ✅
- **验证不通过** → 根据 Claude 的反馈修改产出，重新从第二步开始

## 快速开始

### 环境要求

- **Python 3.9–3.11**（推荐 3.10）
- **Claude Code**（已安装）
- **GPU**（推荐 8GB+ VRAM，CUDA 兼容）
- **Git**

### 启动步骤

```bash
# 1. 进入项目目录
cd C:\Users\Administrator\vsprint

# 2. 查看任务规划
cat docs/superpowers/plans/TASK_PLAN.md

# 3. 查看第一个任务
cat docs/superpowers/tasks/01_环境搭建/01_硬件评估与选择.md

# 4. 开始执行第一个任务
claude -p "请阅读 docs/superpowers/tasks/01_环境搭建/01_硬件评估与选择.md 并完成该任务"

# 5. 任务完成后生成验证方案
python scripts/generate_verify.py tasks/01_环境搭建/01_硬件评估与选择.md

# 6. 执行验证
claude -p "请阅读 docs/superpowers/audit/01_环境搭建/01_硬件评估与选择_verify.md 并验收上一步的任务产出"
```

## 关键技术决策

| 决策点 | 推荐选择 | 理由 |
|--------|----------|------|
| 框架 | PyTorch | 生态丰富，ERes2Net 社区实现在 PyTorch |
| 优化器 | SGD + Momentum | 视觉任务收敛更稳定 |
| 学习率策略 | Cosine Annealing + Warmup | 收敛平滑，效果好 |
| 混合精度 | 开启 | 加速训练，减少显存占用 |
| 部署引擎 | TensorRT（GPU） | NVIDIA GPU 上最优性能 |
| 模型格式 | ONNX | 通用性好，支持多后端 |
