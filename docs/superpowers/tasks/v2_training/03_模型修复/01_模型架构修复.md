## 步骤

### Step 1: 审查当前 dropout 使用情况
当前 `ERes2NetV2.__init__` 接收 `dropout` 参数（通过 **kwargs），但实际未使用：
- `self.dropout = nn.Dropout(p=0.3)` 存在但 p 值硬编码
- config 中 `dropout: 0.4` 未传入

### Step 2: 修复 dropout 参数传递
```python
# 在 ERes2NetV2.__init__ 中添加 dropout 参数
def __init__(self, variant="34", n_mels=80, embedding_dim=192,
             num_speakers=None, scale=4, se_reduction=8,
             pool_attention_dim=128, dropout=0.3):
    ...
    self.dropout = nn.Dropout(p=dropout)
```

### Step 3: 确认 dropout 在 forward 中正确使用
检查 `forward` 方法中 `self.dropout` 的调用位置：
- 应在 embedding_fc 之后、classifier 之前
- 当前代码已有 `self.dropout(x)` 在 bn 之后、embedding_fc 之前，确认位置正确

### Step 4: 更新工厂函数
```python
def create_eres2netv2(variant="34", n_mels=80, embedding_dim=192,
                      num_speakers=None, dropout=0.3, **kwargs):
    return ERes2NetV2(variant=variant, n_mels=n_mels, embedding_dim=embedding_dim,
                      num_speakers=num_speakers, dropout=dropout, **kwargs)
```

### Step 5: 更新 train.py 中的模型创建
在 `train.py` 中传入 dropout 参数：
```python
model = create_eres2netv2(
    ...,
    dropout=model_config.get("dropout", 0.3),
)
```
