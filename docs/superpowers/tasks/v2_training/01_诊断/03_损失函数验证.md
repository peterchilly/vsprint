# Task 1.3: 损失函数正确性验证

**阶段**：01_诊断
**依赖**：无
**预计时间**：1 小时

## 目标
验证 AAM-Softmax 损失函数的数学正确性，排除因损失函数实现 bug 导致的模型坍塌。

## 需要修改/创建的文件
- 创建：`reports/diagnostic/loss_verification.md`
- 创建：`tests/test_losses.py`（测试脚本）

## 步骤

### Step 1: AAM-Softmax 数值测试
构造已知场景验证：
- **场景 A**：embedding 与目标说话人权重完全对齐（cos=1.0），loss 应接近 0
- **场景 B**：embedding 与目标说话人权重正交（cos=0），loss 应较高
- **场景 C**：embedding 与目标说话人权重反向（cos=-1），loss 应最高
- 验证 margin 效果：cos=1.0 时，加 margin 后 loss 应 > 不加 margin

### Step 2: 梯度正确性验证
- 对 embedding 做数值梯度检查（有限差分法）
- 比较解析梯度和数值梯度，误差应 < 1e-4
- 特别检查 margin 区域附近（cos θ > cos(π-m)）的梯度连续性

### Step 3: 权重矩阵检查
- 检查 AAM-Softmax 的 weight 矩阵初始化（Xavier 初始化）
- 检查训练后 weight 的分布是否异常（全零、爆炸等）
- 从 best_model.pth 中提取 criterion_state_dict 检查

### Step 4: 边界条件测试
- batch_size=1 时是否正常
- 单一说话人（所有 label 相同）时是否正常
- 极大 logits 值时是否数值稳定

### Step 5: 撰写验证报告
将所有测试结果汇总到 `reports/diagnostic/loss_verification.md`，包含：
- 每个测试的场景、预期、实际结果
- 结论：损失函数是否正确
- 如发现 bug，描述问题和建议修复

## 产出物

- `tests/test_losses.py` — 可复现的测试脚本
