# Task 1.1: 训练日志审计

**阶段**：01_诊断
**依赖**：无
**预计时间**：30 分钟

## 目标
从训练日志中提取每个 epoch 的指标数据，确认模型坍塌趋势，为后续修复提供数据支撑。

## 需要修改/创建的文件
- 创建：`reports/diagnostic/training_log_analysis.json`

## 步骤

### Step 1: 提取训练日志中的逐 epoch 指标
解析 `checkpoints/training.log`，提取每个 epoch 的：
- EER、minDCF
- pos_mean、neg_mean
- train_loss、train_acc
- learning rate

### Step 2: 标注两轮训练的分界
第一轮训练：epoch 0 ~ ~25（early stop, best EER=0.1347）
第二轮 resume：epoch 26 ~ 71（early stop, best EER=0.1553）

### Step 3: 趋势分析
- 计算 EER 的变化趋势（首 epoch vs 末 epoch）
- 确认 pos_mean 是否持续下降
- 确认 train_acc 是否始终接近随机水平（1/1211≈0.083%）
- 判断是否为模型坍塌（loss 下降但 acc 不升）

### Step 4: 生成 JSON 报告
```json
{
  "round_1": {
    "epochs": [...],
    "eer_trend": "degrading",
    "best_eer": 0.1347,
    "best_epoch": 0
  },
  "round_2": { ... },
  "conclusion": "model_collapse",
  "evidence": [...]
}
```

## 产出物
`reports/diagnostic/training_log_analysis.json` — 包含所有 epoch 指标 + 趋势分析结论

## 验证方法
```powershell
python -c "import json; d=json.load(open('reports/diagnostic/training_log_analysis.json')); print(d['conclusion']); print(len(d['round_1']['epochs']))"
# 预期输出：model_collapse 和 epoch 数量
```

## 通过标准
- [ ] JSON 文件存在且非空
- [ ] 包含两轮训练所有 epoch 的 EER/loss/acc 数据
- [ ] 包含趋势分析结论（model_collapse 或其他）
- [ ] 数据与 training.log 一致
