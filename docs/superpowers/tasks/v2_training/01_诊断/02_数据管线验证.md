# Task 1.2: 数据管线完整性验证

**阶段**：01_诊断
**依赖**：无
**预计时间**：1 小时

## 目标
验证预提取 FBank 特征的数据质量，检查标签对齐、CMVN 状态、说话人分布，排除数据层面的根因。

## 需要修改/创建的文件
- 创建：`reports/diagnostic/data_pipeline_report.json`

## 步骤

### Step 1: 抽样检查特征文件
从 `data/voxceleb/features/` 随机抽取 100 个 .npy 文件：
- 检查 shape（mel 维度必须为 80）
- 检查 dtype（必须为 float32）
- 检查数值范围（log-FBank 应大致 [-20, 5]）
- 检查是否有 NaN/Inf

### Step 2: CMVN 状态验证
- 检查特征是否已做 CMVN（均值接近 0，标准差接近 1）
- 如果未做 CMVN，标记为问题

### Step 3: 标签对齐验证
- 验证 .npy 文件路径 → 说话人 ID 映射是否正确
- 抽取 5 个说话人，检查其所有样本是否在同一 ID 下
- 检查是否有跨说话人的文件错位

### Step 4: 说话人分布统计
- 统计每个说话人的样本数
- 检查最少/最多样本数
- 计算基尼系数或变异系数评估不平衡程度

### Step 5: 生成 JSON 报告
```json
{
  "feature_check": {
    "n_samples": 100,
    "n_mels": 80,
    "dtype": "float32",
    "value_range": [-15.2, 4.8],
    "has_nan": false,
    "has_inf": false
  },
  "cmvn_status": "applied" | "not_applied",
  "label_alignment": "correct" | "misaligned",
  "speaker_distribution": {
    "n_speakers": 1211,
    "min_samples": 3,
    "max_samples": 500,
    "mean_samples": 122,
    "cv": 0.85
  }
}
```

## 产出物
`reports/diagnostic/data_pipeline_report.json`

## 验证方法
```powershell
python -c "import json; d=json.load(open('reports/diagnostic/data_pipeline_report.json')); print(d['cmvn_status']); print(d['label_alignment'])"
# 预期输出：applied/ not_applied 和 correct/misaligned
```
