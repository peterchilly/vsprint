# Task 2.1: 配置一致性修复

**阶段**：02_数据修复
**依赖**：T1.1, T1.2, T1.3
**预计时间**：30 分钟

## 目标
修复 train_config.yaml 中的配置不一致问题，生成 v2 配置文件。

## 需要修改/创建的文件
- 创建：`configs/train_config_v2.yaml`

## 步骤

### Step 1: 复制当前配置
```powershell
cp configs/train_config.yaml configs/train_config_v2.yaml
```

### Step 2: 修复配置项

| 配置项 | 当前值 | 修复为 | 理由 |
|--------|--------|--------|------|
| `data.fixed_length` | 200 | 200（保持） | 2 秒足够，与 data_config 的 300 取统一 |
| `data_config.features.fixed_length` | 300 | 200 | 统一 |
| `training.label_smoothing` | 0.15 | 0.1 | 与 loss.label_smoothing 统一 |
| `loss.label_smoothing` | 0.1 | 0.1（保持） | — |
| `training.amp` | false | true | 开启混合精度加速 |
| `data.num_workers` | 0 | 4 | 并行数据加载 |
| `training.optimizer` | SGD | AdamW | 收敛更稳定 |
| `training.lr` | 0.1 | 0.001 | AdamW 配套 |
| `training.weight_decay` | 0.0005 | 0.05 | AdamW 标准值 |
| `training.batch_size` | 128 | 64 | 减小 batch 防坍塌 |
| `model.dropout` | 0.4 | 0.3 | 适度正则化 |

### Step 3: 同步修改 data_config.yaml
创建 `configs/data_config_v2.yaml`，将 `features.fixed_length` 改为 200。

### Step 4: 验证 YAML 语法
```powershell
python -c "import yaml; c=yaml.safe_load(open('configs/train_config_v2.yaml')); print(c['training']['optimizer'], c['training']['amp'], c['data']['fixed_length'])"
```

## 产出物
- `configs/train_config_v2.yaml` — 修复后的训练配置
- `configs/data_config_v2.yaml` — 修复后的数据配置

## 验证方法
```powershell
python -c "import yaml; c=yaml.safe_load(open('configs/train_config_v2.yaml')); assert c['training']['amp']==True; assert c['training']['optimizer']=='AdamW'; assert c['training']['lr']==0.001; assert c['data']['num_workers']==4; print('Config OK')"
```

- [ ] train_config_v2.yaml 存在且 YAML 语法正确
- [ ] fixed_length 两处统一为 200
- [ ] label_smoothing 统一为 0.1
