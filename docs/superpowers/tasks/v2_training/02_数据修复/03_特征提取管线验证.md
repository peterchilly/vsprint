# Task 2.3: 特征提取管线验证

**阶段**：02_数据修复
**依赖**：T2.1
**预计时间**：1 小时

## 目标
验证预提取 FBank 特征的质量和一致性，确保特征管线无 bug。

## 需要修改/创建的文件
- 创建：`reports/diagnostic/feature_sample_check.json`

## 步骤

### Step 1: 抽样特征文件
从 `data/voxceleb/features/` 随机抽取 10 个说话人，每个 5 个样本，共 50 个 .npy 文件。

### Step 2: 检查特征属性
对每个样本检查：
- **shape**：mel 维度必须为 80，帧数可变
- **dtype**：必须为 float32
- **数值范围**：log-FBank 应大致在 [-20, 5] 范围内
- **NaN/Inf 检查**：不应存在
- **帧数分布**：统计所有样本的帧数分布

### Step 3: 端到端验证
选取一个原始 wav 文件，从 wav → FBank → .npy 完整跑一遍：
- 用 `src/deploy/inference.py` 中的 `extract_fbank` 方法
- 对比结果与已存储的 .npy 文件是否一致

### Step 4: 生成 JSON 报告
```json
{
  "n_samples": 50,
  "n_speakers": 10,
  "feature_shape": {"n_mels": 80, "frame_range": [100, 1500]},
  "dtype": "float32",
  "value_range": {"min": -18.5, "max": 4.2},
  "has_nan": false,
  "has_inf": false,
  "frame_distribution": {"mean": 350, "std": 200, "min": 50, "max": 1200},
  "end_to_end_check": "pass" | "fail",
  "notes": "..."
}
```

## 产出物
`reports/diagnostic/feature_sample_check.json`

## 验证方法
```powershell
python -c "import json; d=json.load(open('reports/diagnostic/feature_sample_check.json')); print(d['n_samples'], d['has_nan'], d['end_to_end_check'])"
# 预期输出：50 False pass
```

## 通过标准
- [ ] JSON 文件存在且非空
- [ ] 包含 50+ 样本的统计信息
- [ ] mel 维度一致为 80
- [ ] 无 NaN/Inf
- [ ] 端到端验证通过
