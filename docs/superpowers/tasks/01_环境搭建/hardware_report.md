## 3. 系统资源检查

| 项目 | 值 | 要求 | 状态 |
|------|-----|------|------|
| CPU | **Intel Core i5-12600KF** | ≥ 8 核 | ⚠️ 10 核，勉强达标 |
| CPU 逻辑线程 | 16 | - | ✅ |
| 内存 | **32 GB** | ≥ 16 GB | ✅ 充足 |
| 磁盘 C:（系统） | 364 GB 总 / 88 GB 可用 | - | ⚠️ 空间偏紧 |
| 磁盘 D:（code） | 100 GB 总 / 6 GB 可用 | - | ❌ **空间严重不足** |
| 磁盘 E: | 954 GB 总 / 208 GB 可用 | - | ✅ 可用 |
| 磁盘 F:（机械硬盘）| 1863 GB 总 / 312 GB 可用 | - | ✅ 可存数据（但速度慢） |
| 可用 SSD 空间 | E: 208 GB | ≥ 200 GB | ✅ 刚好达标 |

**评估结论：**
- CPU 10 核勉强满足要求，数据加载速度可能成为瓶颈（建议 DataLoader `num_workers` 不超过 8）
- 内存 32GB 充足
- **磁盘空间需要注意**：D 盘（代码盘）仅剩 6GB，建议：
  - 将数据集存放到 E: 盘（SSD，208GB 可用）
  - 将模型 checkpoint 存放到 E: 或 F: 盘
  - 清理 D: 盘无用文件
