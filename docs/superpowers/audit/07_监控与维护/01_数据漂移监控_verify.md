## 一、验证目标

| 目标ID | 目标描述 | 优先级 |

| VO-01 | 验证数据漂移监控系统已正确部署并运行 | P0 |

## 二、测试用例

### 2.1 功能测试用例

#### TC-001: 基准数据分布存储验证
| 属性 | 内容 |
|------|------|
| **测试步骤** | 1. 上传训练数据集<br>2. 执行基准分布计算<br>3. 检查存储结果 |
| **预期结果** | - 基准分布统计量正确存储（均值、方差、分位数等）<br>- 存储格式可被监控系统读取<br>- 数据完整性校验通过 |
| **验证方法** | 对比计算结果与预期值，误差<0.01% |

#### TC-002: 线上数据采样验证
| 属性 | 内容 |
|------|------|
| **测试步骤** | 1. 配置采样参数（频率、样本量）<br>2. 触发采样任务<br>3. 验证采样数据 |
| **预期结果** | - 采样频率符合配置（如每小时/每天）<br>- 样本量达到设定阈值<br>- 数据格式与训练数据一致 |
| **验证方法** | 日志检查 + 数据量统计 |

#### TC-003: KS检验正确性验证
| 属性 | 内容 |
|------|------|
| **前置条件** | 准备已知分布的测试数据集 |
| **测试步骤** | 1. 输入两组已知分布数据<br>2. 执行KS检验<br>3. 对比结果 |
| **预期结果** | - KS统计量计算准确<br>- P值计算符合统计学标准<br>- 与scipy.stats.ks_2samp结果一致（误差<1e-6） |
| **验证方法** | 单元测试 + 标准库对比 |

#### TC-004: PSI计算正确性验证
| 属性 | 内容 |
|------|------|
| **前置条件** | 准备分箱配置和测试数据 |
| **测试步骤** | 1. 输入基准分布和当前分布<br>2. 执行PSI计算<br>3. 验证分箱和计算结果 |
| **预期结果** | - 分箱逻辑正确（等频/等宽）<br>- PSI值计算准确<br>- 结果符合行业标准解释（<0.1稳定，0.1-0.25中等，>0.25显著） |
| **验证方法** | 单元测试 + 人工验证 |

#### TC-005: 漂移阈值触发验证
| 属性 | 内容 |
|------|------|
| **前置条件** | 配置阈值：KS_P<0.05 或 PSI>0.25 |
| **测试步骤** | 1. 注入轻微漂移数据（未超标）<br>2. 注入显著漂移数据（超标）<br>3. 检查告警状态 |
| **预期结果** | - 轻微漂移：无告警，记录日志<br>- 显著漂移：触发告警，发送通知<br>- 告警信息包含漂移详情 |
| **验证方法** | 模拟数据注入 + 日志分析 |

#### TC-006: 报告生成验证
| 属性 | 内容 |
|------|------|
| **测试步骤** | 1. 配置报告生成周期<br>2. 等待报告生成或手动触发<br>3. 检查报告内容 |
| **预期结果** | 报告包含以下内容：<br>- ✓ 各特征分布对比图表<br>- ✓ KS检验结果（统计量、P值）<br>- ✓ PSI指标明细<br>- ✓ 漂移特征排名<br>- ✓ 时间趋势图<br>- ✓ 建议措施 |
| **验证方法** | 人工审查 + 自动化检查 |

#### TC-007: 重训练触发验证
| 属性 | 内容 |
|------|------|
| **前置条件** | 配置重训练触发条件 |
| **测试步骤** | 1. 模拟超过阈值的漂移<br>2. 验证触发流程<br>3. 检查重训练任务状态 |
| **预期结果** | - 自动创建重训练任务<br>- 通知相关人员<br>- 记录触发原因和时间<br>- 可追溯执行历史 |
| **验证方法** | 端到端测试 + 任务队列检查 |

### 8.2 参考标准

- **KS检验：** Kolmogorov-Smirnov Test (scipy.stats.ks_2samp)
- **PSI：** Population Stability Index
  - PSI < 0.1: 分布稳定
  - 0.1 ≤ PSI < 0.25: 分布略有变化
  - PSI ≥ 0.25: 分布显著变化
- **显著性水平：** α = 0.05 (KS检验P值阈值)
