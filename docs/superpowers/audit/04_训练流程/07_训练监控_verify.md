# 07_训练监控 — 验证测试设计

> **来源任务：** docs/superpowers/tasks/04_训练流程/07_训练监控.md
> **生成日期：** 2026-06-07

---

## 1. 验证目标概述

本任务的目标是确保训练过程中的关键指标可观测、异常可告警、日志可管理。验证核心包括：

1. **训练看板** — 实时展示 loss 曲线、学习率变化曲线、GPU 利用率
2. **异常告警** — 当训练出现 loss NaN、梯度爆炸等异常时能够及时告警
3. **日志管理** — 训练日志有合理的存储、轮转和清理机制

---

## 2. 测试用例设计

### TC-01: 训练看板功能验证
- **前置条件：** 训练脚本已配置 TensorBoard 或 WandB 等可视化工具
- **测试步骤：**
  1. 启动训练脚本（可运行 2-3 个 epoch 即可）
  2. 打开 TensorBoard / WandB 看板
  3. 检查是否能正常显示以下指标：
     - 训练 loss 曲线（train_loss）
     - 验证 loss 曲线（val_loss）
     - 学习率变化曲线（learning_rate）
     - GPU 利用率 / 显存占用（如配置了 nvidia-smi 集成）
- **预期结果：** 所有指标在看板中正常显示，曲线随 epoch 更新

### TC-02: Loss NaN 检测告警
- **前置条件：** 异常检测机制已配置
- **测试步骤：**
  1. 模拟或构造一个会导致 loss 为 NaN 的场景（如使用异常大的学习率）
  2. 启动训练
  3. 检查是否触发 NaN 告警
  4. 检查告警方式（日志输出 / 文件写入 / 通知）
- **预期结果：** 系统在检测到 loss 为 NaN 时，输出明确的告警信息并记录到日志

### TC-03: 梯度爆炸检测告警
- **前置条件：** 梯度监控已配置
- **测试步骤：**
  1. 关闭梯度裁剪（或设置极高的 max_norm）
  2. 启动训练，观察梯度范数（gradient norm）
  3. 当梯度范数超过预设阈值（如 100.0）时，检查是否触发告警
- **预期结果：** 系统在梯度范数超过阈值时输出告警，记录异常梯度值

### TC-04: 日志存储与轮转
- **前置条件：** 日志系统已配置
- **测试步骤：**
  1. 启动训练，运行至少 5 个 epoch
  2. 检查日志文件是否正常写入
  3. 检查日志格式是否包含时间戳、epoch、loss、lr 等关键信息
  4. 检查日志轮转机制：
     - 单文件超过大小限制（如 50MB）后是否自动轮转
     - 旧日志是否保留（如保留最近 7 天）
- **预期结果：** 日志正常写入，格式规范，轮转机制按配置生效

### TC-05: GPU 利用率监控
- **前置条件：** GPU 监控集成已配置
- **测试步骤：**
  1. 启动训练
  2. 检查是否记录以下 GPU 指标：
     - GPU 利用率（%）
     - 显存占用（MB）
     - 温度（°C）
  3. 验证指标采样频率是否合理（建议每 30-60 秒采样一次）
- **预期结果：** GPU 指标正常记录，可用于分析训练效率瓶颈

## 3. 验证方法与步骤

### 3.1 代码审查
1. 检查训练主循环中是否包含监控逻辑
2. 确认 `torch.cuda.amp.GradScaler` 的 `get_scale()` 或 `check_scale()` 是否用于检测 NaN
3. 确认梯度范数计算：`torch.nn.utils.clip_grad_norm_` 或手动计算 `param.grad.norm()`
4. 确认日志写入使用 `logging` 模块或专用框架

### 3.2 实际运行测试
1. 使用小规模数据集（如 100 张随机图片）快速验证训练流程
2. 运行 3-5 个 epoch，验证监控数据正常产生
3. 构造异常场景（大学习率）验证告警

### 3.3 日志检查
```bash
# 查看最近的训练日志
cat logs/train_*.log | tail -100

# 检查 TensorBoard 事件文件
ls runs/

# 查看 GPU 日志（如果有）
cat logs/gpu_*.log | tail -50
```

## 6. 风险点与注意事项

| 风险点 | 影响 | 缓解措施 |
|--------|------|----------|
| TensorBoard 未正确写入事件文件 | 看板无数据 | 训练前检查 `SummaryWriter` 是否正确初始化 |
| NaN 检测延迟 | 可能已经训练了多个无效 epoch 才发现问题 | 每个 step 都检查 loss，而不是每个 epoch |
| 日志文件过大占满磁盘 | 训练中断 | 配置日志轮转（rotation），限制总大小 |
| GPU 监控采样过频 | 增加系统开销 | 采样间隔 ≥30 秒 |
| 告警信息过于简略 | 无法定位问题 | 告警应包含 epoch、step、loss 值、梯度范数等上下文 |
