# ERes2NetV2 说话人识别 Pipeline — 任务规划文档

> **目标**：基于 ERes2NetV2 模型架构，规划从零到部署的完整说话人识别训练和部署 pipeline。
> **适用场景**：说话人验证（Speaker Verification）/ 说话人识别（Speaker Identification），使用 VoxCeleb 数据集。
> **文档版本**：v2.0（Speaker Recognition）

---

## 总览：Pipeline 七大阶段

```
环境搭建 → 数据准备 → 模型构建 → 训练流程 → 评估与测试 → 部署与推理 → 监控与维护
```

---

## 阶段二：数据准备（说话人识别 / VoxCeleb）

### 目标
构建高质量、规范化的训练/验证/测试音频数据集，用于说话人识别任务。

### 具体任务拆解

#### T2.1 数据集选择与获取
- [ ] 确定数据集：**VoxCeleb1**（首选，1,211 说话人，~15 万条语音）
- [ ] 可选扩展：**VoxCeleb2**（5,994 说话人，~100 万条语音）
- [ ] 可选：**MUSAN 噪声库**（用于数据增强）
- [ ] 下载数据集：从官方源或 Hugging Face Datasets 下载
- [ ] 验证数据完整性：文件数量、音频格式（.wav/.ogg/.m4a）、采样率、说话人标签一致性

#### T2.2 数据探索与分析（EDA）
- [ ] 统计每个说话人的语音片段数量，检查说话人分布不平衡
- [ ] 分析音频时长分布（最短/最长/平均/中位数）
- [ ] 分析采样率、声道数、音频格式一致性
- [ ] 可视化样本的波形图和频谱图
- [ ] 检查损坏文件、静音片段
- [ ] 记录数据集元信息（说话人数、总样本数、平均时长、采样率）

#### T2.3 数据预处理与增强
- [ ] **基础处理**：
  - [ ] 音频重采样到统一采样率（16kHz）
  - [ ] 转换为单声道（Mono）
  - [ ] 预加重（Pre-emphasis）
  - [ ] 提取声学特征：**FBank（80 维 Filter Bank）**
    - 帧长 25ms，帧移 10ms，Hamming 窗
- [ ] **训练集增强**（提升鲁棒性）：
  - [ ] 加性噪声：RIR（房间脉冲响应）+ MUSAN 噪声
  - [ ] 速度扰动：0.9x / 1.0x / 1.1x
  - [ ] SpecAugment：时间遮蔽 + 频率遮蔽
  - [ ] 随机增益：音量扰动（±5dB）
  - [ ] 时长截断/填充（统一输入长度，如 2 秒）
- [ ] **验证/测试集处理**：
  - [ ] 仅基础处理（重采样 → 单声道 → FBank）
  - [ ] 不做随机增强，确保评估稳定性
  - [ ] 可选：CMVN 归一化

#### T2.4 数据集划分
- [ ] 使用 VoxCeleb 官方划分：
  - **VoxCeleb1 Dev** → 训练集（1,211 说话人）
  - **VoxCeleb1 Test** → 测试集（40 说话人 + trials 文件）
  - 或从 Dev 中划分一部分作为验证集
- [ ] 确保说话人身份在训练集和测试集之间不重叠
- [ ] 保存划分索引（JSON/CSV），设置随机种子
- [ ] 生成 trials 文件格式的评估列表（enrollment / test 对）

#### T2.5 DataLoader 构建
- [ ] 继承 `torch.utils.data.Dataset`，实现 `__len__` 和 `__getitem__`
  - 返回：FBank 特征 tensor + 说话人 label（整数 ID）
- [ ] 实现说话人标签编码（string → int 映射）
- [ ] 配置 DataLoader：
  - [ ] `batch_size`：根据 GPU 显存调整（如 64/128）
  - [ ] `num_workers`：4–8
  - [ ] `pin_memory=True`
  - [ ] `shuffle=True`（仅训练集）
  - [ ] `drop_last=True`（训练集）
- [ ] 采样策略：N-way K-shot 或随机采样 + CrossEntropyLoss

- ✅ 划分好的训练/验证/测试音频数据集
- ✅ FBank 特征提取 pipeline
- ✅ 数据增强策略确定

## 阶段三：模型构建

### 目标
基于 ERes2NetV2 架构实现完整的模型代码，包括 Backbone、分类头及必要的辅助模块。

### 具体任务拆解

#### T3.1 ERes2NetV2 Backbone 实现
- [ ] **基础组件**：
  - [ ] 实现 `ConvBNReLU`（卷积 + BatchNorm + ReLU 基础块）
  - [ ] 实现 1×1 卷积层（用于通道维度变换：降维和升维）
  - [ ] 实现 Batch Normalization 层
- [ ] **ERes2NetV2 Block 核心模块**：
  - [ ] **分组特征提取**：将输入特征图按通道分为多组（如 4 组），实现层级化特征处理
  - [ ] **特征重用机制**：后续分组复用前面分组的输出（`y_i = conv(x_i + y_{i-1})`），避免重复计算
  - [ ] **通道交互模块**：实现轻量级注意力机制（全局平均池化 → FC → 通道权重 → 特征重校准）
  - [ ] **选择性残差融合**：引入可学习权重 α, β 实现 `output = α·F(x) + β·x`
- [ ] **网络层级结构**：
  - [ ] Stem 层（初始卷积 + 池化）
  - [ ] Stage 1–4（逐步降采样，通道数递增）
  - [ ] 每 Stage 内部堆叠若干 ERes2NetV2 Block
  - [ ] 全局平均池化 + 全连接分类头

#### T3.2 模型变体配置
- [ ] 实现不同深度的模型变体（类比 ResNet 的 50/101/152 层）：
  - [ ] `ERes2NetV2-50`（轻量级）
  - [ ] `ERes2NetV2-101`（标准版）
  - [ ] `ERes2NetV2-152`（深度版）
- [ ] 使用配置文件管理不同变体的层数、通道数、分组数等超参数

#### T3.3 预训练权重支持
- [ ] 加载 ImageNet 预训练权重（如果有公开的 ERes2NetV2 权重）
- [ ] 实现权重加载逻辑：
  - [ ] 全量加载（fine-tuning 场景）
  - [ ] 部分加载（仅加载 Backbone，分类头随机初始化）
- [ ] 权重冻结策略：初期冻结 Backbone 浅层，逐步解冻

#### T3.4 模型验证
- [ ] 检查模型输出维度是否符合预期
- [ ] 打印模型结构，确认层数和参数量
- [ ] 前向传播测试：随机输入验证无报错
- [ ] 参数量统计：对比预期值，确认无冗余
- [ ] 模型保存/加载测试：验证 checkpoint 可正确恢复

#### T3.5 辅助模块
- [ ] 混合精度训练支持（AMP / `torch.cuda.amp`）
- [ ] FLOPs 计算工具（`fvcore` / `thop`）用于评估计算复杂度
- [ ] 模型可视化：生成网络结构图或 ONNX 格式查看

### 阶段交付物
- ✅ ERes2NetV2 模型代码完整实现
- ✅ 多种变体配置
- ✅ 预训练权重加载功能

## 阶段四：训练流程

### 目标
设计并执行高效的训练流程，使模型在训练集上收敛并达到最优性能。

### 具体任务拆解

#### T4.1 损失函数设计
- [ ] **分类任务**：交叉熵损失（CrossEntropyLoss）
  - [ ] 类别不平衡场景：带权重的交叉熵 或 Focal Loss
- [ ] **辅助损失**（可选）：
  - [ ] Label Smoothing（防止过拟合，默认 0.1）
  - [ ] 知识蒸馏损失（如果有教师模型）

#### T4.2 优化器配置
- [ ] 选择优化器：
  - [ ] **SGD + Momentum**（推荐，稳定收敛）：momentum=0.9, weight_decay=1e-4
  - [ ] **AdamW**（适合快速实验）：weight_decay=0.05
- [ ] 学习率策略：
  - [ ] Warmup：前 5 个 epoch 线性增长到目标学习率
  - [ ] Cosine Annealing 或 StepLR 衰减
  - [ ] 初始学习率：SGD 设为 0.1（batch=256），按比例缩放

#### T4.3 训练循环实现
- [ ] 标准训练流程：
  1. [ ] 设置模型为 `train()` 模式
  2. [ ] 遍历 DataLoader，逐 batch 前向传播
  3. [ ] 计算损失，反向传播
  4. [ ] 梯度裁剪（防止梯度爆炸，max_norm=1.0）
  5. [ ] 优化器 step
  6. [ ] 记录训练指标（loss、accuracy）
- [ ] 混合精度训练集成（`torch.cuda.amp.autocast` + `GradScaler`）
- [ ] 梯度累积（显存不足时，累积多个 batch 后更新权重）

#### T4.4 验证流程
- [ ] 每个 epoch 结束后执行验证：
  1. [ ] 设置模型为 `eval()` 模式
  2. [ ] 关闭梯度计算（`torch.no_grad()`）
  3. [ ] 遍历验证集，计算 Top-1 / Top-5 准确率
  4. [ ] 记录验证指标
- [ ] 模型保存策略：
  - [ ] 保存验证集最优模型（`best_model.pth`）
  - [ ] 定期保存 checkpoint（每 N 个 epoch）
  - [ ] 保存最新模型（`last_model.pth`）

#### T4.5 训练策略
- [ ] **两阶段训练**：
  - [ ] **阶段一（Warm-up）**：冻结 Backbone 浅层，仅训练分类头（3–5 epochs）
  - [ ] **阶段二（Fine-tuning）**：解冻全部参数，端到端训练
- [ ] **渐进式训练**：
  - [ ] 从小分辨率开始（如 128×128），逐步增大到目标分辨率（224×224）
- [ ] **早停策略**：
  - [ ] 验证集准确率连续 N 个 epoch 不提升则停止训练
  - [ ] 学习率衰减后仍未提升则终止

#### T4.6 训练超参数配置
- [ ] 编写配置文件（YAML/JSON）管理所有超参数：
  ```yaml
  training:
    epochs: 100
    batch_size: 64
    lr: 0.025
    optimizer: SGD
    momentum: 0.9
    weight_decay: 0.0001
    lr_schedule: cosine
    warmup_epochs: 5
    label_smoothing: 0.1
    grad_clip: 1.0
    amp: true
  ```

#### T4.7 训练监控
- [ ] 实时监控训练/验证 loss 曲线
- [ ] 监控学习率变化曲线
- [ ] 监控 GPU 利用率、显存占用
- [ ] 设置告警：训练异常（loss NaN、梯度爆炸）时通知

### 阶段交付物
- ✅ 完整训练循环代码
- ✅ 训练配置文件
- ✅ 训练日志和可视化看板

## 阶段五：评估与测试

### 目标
对训练好的模型进行全面评估，确保其在未见数据上的泛化能力和鲁棒性。

### 具体任务拆解

#### T5.1 标准评估指标
- [ ] **图像分类**：
  - [ ] Top-1 准确率
  - [ ] Top-5 准确率
  - [ ] 每类别精确率、召回率、F1 分数
- [ ] **混淆矩阵**：分析模型在哪些类别上容易混淆
- [ ] **PR 曲线与 ROC 曲线**（二分类场景）

#### T5.2 测试集评估
- [ ] 在**完全未见**的测试集上进行最终评估
- [ ] 使用 **Test-Time Augmentation (TTA)**：
  - [ ] 多尺度测试
  - [ ] 水平翻转测试
  - [ ] 综合预测结果（平均或投票）
- [ ] 报告最终性能指标

#### T5.3 鲁棒性评估
- [ ] **噪声鲁棒性**：在测试图片上添加高斯噪声、椒盐噪声后评估
- [ ] **尺寸鲁棒性**：不同分辨率输入下的性能变化
- [ ] **对抗鲁棒性**（可选）：FGS/PGD 攻击下的表现
- [ ] **跨域泛化**：不同数据集间的迁移性能

#### T5.4 效率评估
- [ ] **推理速度**：
  - [ ] 单张图片推理延迟（ms）
  - [ ] 吞吐量（FPS，batch=1 和 batch=N）
- [ ] **计算复杂度**：
  - [ ] FLOPs（浮点运算次数）
  - [ ] 参数量（Params）
- [ ] **资源占用**：
  - [ ] GPU 显存峰值
  - [ ] CPU 内存占用
- [ ] 对比基线模型（ResNet50、ERes2Net 等）的性能和效率

#### T5.5 消融实验
- [ ] 验证 ERes2NetV2 各创新点的贡献：
  - [ ] w/o 特征重用 → 对比准确率下降
  - [ ] w/o 通道交互 → 对比准确率下降
  - [ ] w/o 选择性融合（改为标准残差）→ 对比准确率下降
  - [ ] 不同分组数（scale=2/4/6）→ 对比性能与效率 trade-off
- [ ] 记录消融实验结果，形成实验报告

#### T5.6 错误分析
- [ ] 收集模型预测错误的样本
- [ ] 分析错误模式：
  - [ ] 标注错误？
  - [ ] 样本质量差（模糊、遮挡）？
  - [ ] 类别间相似度高？
- [ ] 根据分析结果提出改进方向

### 阶段交付物
- ✅ 测试集评估报告（含所有指标）
- ✅ 混淆矩阵和可视化
- ✅ 消融实验结果

## 阶段六：部署与推理

### 具体任务拆解

#### T6.1 模型导出与优化
- [ ] **模型格式转换**：
  - [ ] PyTorch → ONNX（导出为标准格式）
  - [ ] ONNX → TensorRT（NVIDIA GPU 加速）
  - [ ] 或导出为 TorchScript（JIT 编译）
- [ ] **模型量化**：
  - [ ] FP32 → FP16（半精度，推理速度提升约 2x）
  - [ ] INT8 量化（进一步加速，需校准数据集）
  - [ ] 对比量化前后的精度损失
- [ ] **模型剪枝**（可选）：
  - [ ] 通道剪枝（去除不重要的通道）
  - [ ] 结构化剪枝（保持硬件友好）
  - [ ] 剪枝后 fine-tune 恢复精度

#### T6.2 推理引擎选型
- [ ] **GPU 部署**：

  - [ ] ONNX Runtime

## 附录：关键技术决策点

| 决策点 | 选项 | 推荐选择 | 理由 |
|--------|------|----------|------|
| 框架 | PyTorch / TensorFlow | PyTorch | 生态丰富，ERes2Net 社区实现在 PyTorch |
| 优化器 | SGD / AdamW | SGD + Momentum | 说话人识别任务收敛更稳定 |
| 学习率策略 | StepLR / Cosine | Cosine Annealing + Warmup | 收敛平滑，效果好 |
| 混合精度 | 开启 / 关闭 | 开启 | 加速训练，减少显存占用 |
| 部署引擎 | TensorRT / ONNX Runtime | TensorRT（GPU）| NVIDIA GPU 上最优性能 |
| 模型格式 | ONNX / TorchScript | ONNX | 通用性好，支持多后端 |
| 音频特征 | MFCC / FBank / Mel | 80 维 FBank | 说话人识别标准特征 |
| 损失函数 | Softmax / AAM-Softmax / ArcFace | AAM-Softmax | 说话人识别主流损失 |
| 评估指标 | Accuracy / EER / minDCF | EER + minDCF | 说话人验证标准指标 |
