## 0. 全局视角：这个工程到底在做什么

### 0.1 任务定义

这是一个 **说话人识别（Speaker Recognition）** 工程，更准确地说是 **说话人验证（Speaker Verification）**：

- 给定一段语音 → 提取一个固定维度的向量，称为 **说话人嵌入（embedding）**。
- 判断两段语音是否来自同一个人 → 比较两个 embedding 的相似度（余弦相似度）。

与「语音识别（ASR，识别说了什么字）」完全不同，这里只关心「是谁说的」，不关心内容。

### 0.2 整体数据流

```
原始 WAV 音频
   │  scripts/extract_features.py
   ▼
FBank 特征 (.npy)            ← 把变长波形变成定长频谱矩阵
   │  src/datasets/speaker_dataset.py
   ▼
(batch, 1, n_mels, frames)   ← DataLoader 喂给模型
   │  src/models/eres2netv2.py
   ▼
embedding (batch, 192)        ← 说话人向量
   │  src/models/losses.py (AAM-Softmax)
   ▼
分类损失 / 反向传播            ← 训练
   │  src/training/trainer.py
   ▼
用 embedding 算 EER / minDCF  ← 验证（说话人验证标准指标）
```

### 0.3 工程文件地图（实际入口是根目录 `train.py`）

| 文件 | 作用 |
|------|------|

## 1. 数据准备：WAV → FBank 特征

代码：`scripts/extract_features.py`

### 1.1 为什么要离线提取特征（而不是训练时实时算）

**知识点**：FBank（Mel 滤波器组）特征提取涉及 FFT、Mel 滤波、对数压缩，是 CPU 密集型操作。

**为什么离线**：
- 训练 80 epoch × 13 万条音频，每条都要重复算 FBank → 浪费大量 CPU。
- 一次提取成 `.npy`，后续训练直接 `np.load`，I/O 便宜。
- 这是语音工程的**标准做法**：特征提取与训练解耦。

**我的思考**：这本质上是「把不变的计算结果缓存下来」。代价是磁盘占用（约 35GB），但换来训练速度数量级提升，值得。

### 1.2 音频预处理参数（`data_config.yaml`）

```yaml
sample_rate: 16000      # 16kHz 单声道
frame_length: 400       # 帧长 25ms = 0.025×16000
frame_shift: 160        # 帧移 10ms = 0.010×16000
n_fft: 512
n_mels: 80
f_min: 0.0, f_max: 8000.0
preemphasis: 0.97
```

逐个解释每个知识点：

#### (1) 重采样到 16kHz
- 电话语音通常 8kHz/16kHz，VoxCeleb 来自视频，原始多为 16kHz。
- 统一到 16kHz 保证所有音频频率上限一致（奈奎斯特频率 8kHz）。
- **为什么不用更高**：说话人信息主要在 4kHz 以下，16kHz 足够，省一半算力。

#### (2) 预加重（preemphasis=0.97）
```python
def apply_preemphasis(self, audio):
    return np.append(audio[0], audio[1:] - self.preemphasis * audio[:-1])
```
- 语音信号高频能量天然低（频谱以 ~6dB/oct 衰减）。
- 预加重是一阶高通滤波器，抬升高频，**平衡频谱**，让后续 FBank 各频带能量更均衡。
- **为什么 0.97**：经验值，业界标准。现代端到端模型有时省略，但保留无害。

#### (3) 分帧：帧长 25ms / 帧移 10ms
- 语音是**短时平稳**信号：在 20~30ms 内可近似看作 stationary，才能做 FFT。
- 帧移 10ms < 帧长 25ms → 帧间有重叠，避免帧边界信息丢失。
- **为什么 25/10ms**：语音处理几十年来的「黄金参数」，几乎所有说话人/ASR 系统都用。

#### (4) FBank（Mel 滤波器组）—— 核心特征
```python
fbank = librosa.feature.melspectrogram(
    y=audio, sr=sr, n_fft=512, hop_length=160,
    win_length=400, n_mels=80, fmin=0, fmax=8000, power=2.0)
log_fbank = np.log(np.clip(fbank, 1e-10, None))  # 对数压缩
```

**知识点链**：
1. **FFT**：把每一帧从时域变到频域，得到功率谱。
2. **Mel 滤波器组**：用一组三角滤波器对功率谱按 **Mel 频率**求和。
   - **Mel 刻度**：人耳对低频分辨率高、高频分辨率低；Mel 刻度在低频线性、高频对数，**模拟人耳听觉**。
   - **为什么用 Mel**：说话人特征最终要靠人耳/语义区分，用听觉刻度比线性频率更合理。
3. **对数压缩**：人耳对响度的感知是对数的；且对数把乘性动态范围变成加性，稳定数值。
4. **80 维**：说话人识别行业标准维度（Kaldi、WeSpeaker 等都用 80）。

**为什么选 FBank 而不是 MFCC**：
- MFCC 是在 FBank 上再做 DCT 得到去相关的系数，适合 GMM-HMM 时代（高斯假设不相关）。
- 深度网络（CNN）本身能学习特征相关性，**不需要 DCT 去相关**，反而 FBank 保留了更多信息。
- 所以现代声纹模型清一色用 **log-FBank**。

**我的思考**：FBank 把「一段变长波形」变成了「(时间帧 × 80 频带) 的矩阵」，可以像图像一样丢进 CNN。这就是为什么接下来用 2D 卷积。

### 1.3 一个关键的不一致点（需要指出）

- `data_config.yaml` 写 `use_energy: false`，注释说「只用 80 维 FBank」。
- 但 `extract_features.py` 的 `extract_fbank` **默认 `use_energy=True`**，会在 80 维后**拼 1 维能量 → 81 维**。
- `docs/VERIFICATION_REPORT.md` 第 149 行也确认实际是「80 维 + 1 维能量 = 81 维」。

→ 实际存盘的是 **81 维**特征。`train.py` 里通过 `sample_feat.shape[1]` 动态读取真实维度（81）传给模型，所以**不会报错**，但 `train_config.yaml` 里 `n_mels: 80` 与实际不符，是个埋雷。建议统一关掉能量维或在配置里写 81。

## 2. 数据集与 DataLoader

代码：`src/datasets/speaker_dataset.py`，根 `train.py` 中的 `SplitSpeakerDataset`。

### 2.1 目录结构与文件列表构建

数据按 VoxCeleb 的 `idXXXX/video_id/0000N.npy` 嵌套存放。`_build_file_list` 用 `rglob("*.npy")` 递归收集所有特征文件，并建立：

```python
speaker_to_id: {说话人名 -> 整数 ID}
labels: [每个样本对应的说话人 ID]
```

**知识点**：说话人识别是**多分类**任务，每个说话人一个类别。整数 ID 是分类标签。

### 2.2 固定长度填充/截断（`_pad_or_truncate`）

```python
def _pad_or_truncate(self, fbank, target_frames):
    if n_frames >= target_frames: return fbank[:target_frames]
    else:  # 重复填充
        repeats = (target_frames // n_frames) + 1
        fbank = np.tile(fbank, (repeats, 1))
        return fbank[:target_frames]
```

**知识点**：CNN（尤其带 BatchNorm 和池化）通常要求 batch 内形状一致，才能 `torch.stack`。

- **截断**：长音频取前 `target_frames` 帧（约 2~3 秒）。说话人音色在 2~3 秒内已足够区分。
- **重复填充**：短音频循环复制到目标长度，而不是零填充。

**为什么重复而不是补零**：
- 补零会让 FBank 出现大段「静音帧」，干扰统计池化（尤其是 attentive pooling 的注意力）。
- 重复保留语音内容，至少不会引入虚假静音段。
- **我的思考**：这是工程折中。更严谨的做法是随机截取片段 + 对短音频做 N-way 拼接，但重复填充实现简单、够用。

### 2.3 维度约定

`.npy` 存的是 `(n_frames, n_mels)`，`__getitem__` 里转置成 `(n_mels, n_frames)` 再返回。模型再 `unsqueeze(1)` 成 `(batch, 1, n_mels, frames)`。

**为什么是 (n_mels, frames) 而不是反过来**：把频率维度当图像「高」、时间维度当「宽」，2D 卷积在频-时平面上滑动，符合直觉。

### 2.4 按说话人划分训练/验证集（关键正确性点）

`train.py::create_split` 和 `split_dataset_by_speakers` 都按**说话人**划分：

```python
val_dirs, train_dirs = shuffled[:val_count], shuffled[val_count:]
```

**知识点 / 为什么必须按说话人划分**：
- 如果随机按样本划分，**同一个人的语音会同时出现在训练集和验证集**。
- 模型可能学到「记忆」该说话人的录音环境/麦克风特征，而非音色本身 → 验证指标虚高。
- 按说话人切分保证「验证集说话人训练时没见过」，这才是对泛化能力的真实检验。

**我的思考**：这是说话人识别最容易踩的坑。这个工程做对了。

### 2.5 N-way K-shot 采样器（备选）

`NWayKShotSampler`：每个 batch 含 N 个说话人 × 每人 K 条语音。这是为「度量学习/原型网络」准备的采样方式，本工程默认未启用（`n_way: null`）。保留它是为了将来做 few-shot 验证实验。

## 3. 模型：ERes2NetV2 Backbone

代码：`src/models/eres2netv2.py`

### 3.1 整体结构

```
输入 (batch, 1, n_mels, frames)
  │
  ├─ Stem: Conv3x3 + BN + ReLU + MaxPool        # 初始特征提取 + 降采样
  │
  ├─ Stage1~4: ERes2NetV2 BasicBlock 堆叠        # [3,4,6,3] 个 block（34层版）
  │   每个 block：1×1投影 → 分组特征提取+复用 → SE注意力 → 选择性残差融合
  │
  ├─ 频率维平均: (B,C,F,T) → (B,C,T)            # 压掉频率维
  ├─ AttentiveStatsPool: (B,C,T) → (B,2C)        # 注意力加权 均值+标准差
  ├─ BatchNorm1d → Dropout → Linear(2C→192)      # 得到 embedding
  └─ Linear(192→num_speakers) [可选]              # 分类头（仅训练用）
```

输出：`embedding (B,192)` + `logits (B, num_speakers)`。

### 3.2 Stem（卷积头）

```python
self.stem = nn.Sequential(
    nn.Conv2d(1, 64, 3, stride=1, padding=1, bias=False),
    nn.BatchNorm2d(64), nn.ReLU(inplace=True),
    nn.MaxPool2d(kernel_size=3, stride=2, padding=1))
```

**知识点**：
- **Conv2d(1→64)**：输入 1 通道（单张 FBank 谱图），输出 64 个特征图。
- **MaxPool stride=2**：空间降采样一半，减少后续计算量、扩大感受野。
- **为什么 stem 用小卷积**：3×3 小卷积参数少、感受野可堆叠，是 ResNet 系列的标准开场。

### 3.3 ERes2NetV2 BasicBlock —— 核心创新

这是整个模型区别于普通 ResNet 的地方。展开看：

```python
class ERes2NetV2BasicBlock(nn.Module):
    def __init__(self, in_channels, mid_channels, stride=1, scale=4, ...):
        # 1×1 投影（处理通道数 + 空间降采样）
        self.proj = ConvBNReLU(in_channels, mid_channels, kernel_size=1, stride=stride)
        # scale 个分组 3×3 卷积
        self.convs = ModuleList([ConvBNReLU(group_ch, group_ch, 3, 1, 1) for i in range(scale)])
        # 通道注意力
        self.se = SELayer(mid_channels, reduction=se_reduction)
        # 可学习残差权重
        self.alpha = nn.Parameter(torch.ones(1))
        self.beta  = nn.Parameter(torch.ones(1))
```

forward 的核心是「分组特征提取 + 特征复用」：

```python
out = self.proj(x)                          # (B, mid, F, T)
xs = torch.chunk(out, self.scale, dim=1)    # 沿通道切成 4 份
ys = []
for i in range(self.scale):
    if i == 0:      y_i = xs[i]                     # 第1组直接透传
    elif i == 1:    y_i = self.convs[i](xs[i])      # 第2组单独卷积
    else:           y_i = self.convs[i](xs[i] + ys[-1])  # 第3,4组：输入+上一组输出
    ys.append(y_i)
out = torch.cat(ys, dim=1)                 # 拼回 (B, mid, F, T)
out = self.se(out)                         # 通道注意力重校准
out = self.alpha * out + self.beta * identity  # 选择性残差融合
```

#### 知识点 1：Res2Net 的「分组 + 复用」
- 普通 ResNet：一个 block 里串行 3×3 卷积，所有通道一起算。
- Res2Net：把通道**切成 scale 份**，第 i 份的输出 = 卷积(第 i 份输入 **+** 第 i-1 份输出)。
- 效果：形成**层级化感受野**，第 4 组经过的卷积次数最多 → 感受野最大。一个 block 内就有了**多尺度特征**。

**比喻**：接力赛，后一棒在前一棒基础上继续跑，而不是各跑各的。

#### 知识点 2：特征重用（ERes2NetV2 的改进）
- `y_i = self.convs[i](xs[i] + ys[-1])` 中 `ys[-1]` 就是「复用前一组结果」。
- 后续分组复用前面分组输出 → **避免重复计算**，参数效率更高。

#### 知识点 3：SE 通道注意力（`SELayer`）
```python
se = gap(x)                          # 全局平均池化 → (B,C,1,1)
se = sigmoid(fc2(relu(fc1(se))))     # 学习每个通道的重要性权重
return x * se                        # 逐通道重新加权
```
- **为什么需要**：不同频带/特征通道对说话人辨识的贡献不同（基频、共振峰等更关键）。SE 让网络学会「哪些通道重要，加强它们」。
- **为什么轻量**：先用 1×1 卷积降维到 C/r，再升回 C，参数极少。

#### 知识点 4：选择性残差融合（`alpha * out + beta * identity`）
- 普通 ResNet：`out = F(x) + x`，残差权重恒为 1。
- 这里用**可学习标量** α、β 控制主干与跳跃连接的混合比例。
- **为什么**：不同深度、不同任务下，「恒等映射」和「变换」的最佳比例不同，让网络自己学比写死更灵活。
- **我的思考**：α、β 初始化为 1（等价于标准残差），训练中自适应调整，是个低成本高收益的改进。

### 3.4 AttentiveStatsPool（注意力统计池化）—— 说话人识别的关键

```python
class AttentiveStatsPool(nn.Module):
    def forward(self, x):              # x: (B, C, T)
        weights = self.attention(x)    # 每帧的重要性权重 (B,C,T)
        mean = sum(x * weights, dim=2)                 # 注意力加权均值
        std  = sqrt(sum((x*weights - mean)^2)/T)       # 注意力加权标准差
        return cat([mean, std], dim=1)  # (B, 2C)
```

**知识点 / 为什么这是声纹模型的核心**：
- 变长语音经过 CNN 后是 `(B, C, T)`，T 随输入长度变化。要得到固定维度 embedding，必须把 T 维「压掉」。
- 最朴素：全局平均池化。但**不是每一帧都同等重要**——静音帧、噪声帧应被忽略，含说话人特征信息的帧应被加权。
- **AttentiveStatsPool** 用注意力自动学「哪几帧重要」，并输出 **均值 + 标准差**。
  - 均值：音色主体。
  - 标准差：音色变化/动态信息。
  - 拼两者 → 比只取均值表达力更强。这是 ECAPA-TDNN、ERes2Net 等现代声纹模型的**标准池化**。

**我的思考**：这一层是「从序列到向量」的桥梁，决定了 embedding 质量。本工程用了正确的池化方式。

### 3.5 Embedding 头

```python
self.bn = nn.BatchNorm1d(final_channels * 2)
self.dropout = nn.Dropout(p=0.3)
self.embedding_fc = nn.Linear(final_channels * 2, embedding_dim)  # →192
```

- **BN1d**：稳定 embedding 分布。
- **Dropout**：防过拟合。
- **降到 192 维**：192 是说话人 embedding 的常见维度（与 ResNet34-16k 等开源模型对齐，便于复用预训练权重/做迁移）。

### 3.6 分类头（可选）

```python
self.classifier = nn.Linear(embedding_dim, num_speakers) if num_speakers else None
```

**重要陷阱（见 §9 诊断）**：当用 AAM-Softmax 损失时，分类损失**不经过** `self.classifier`，而是用 `AAMSoftmaxLoss` 内部自己的权重矩阵。所以 `self.classifier` 的 logits **不会被训练**。

### 3.7 权重初始化

```python
Conv2d: normal_(0, sqrt(2/n))   # Kaiming 初始化
BatchNorm: weight=1, bias=0
Linear: normal_(0, 0.01)
```

**为什么 Kaiming**：配合 ReLU 的方差守恒推导，保证前向激活方差不爆炸/消失，深层网络能收敛。这是 ResNet 能训深的基石之一。

## 4. 损失函数：AAM-Softmax

代码：`src/models/losses.py`

### 4.1 为什么不能用普通 CrossEntropy

普通 softmax 把各类别映射成概率，但**不保证类内紧凑、类间分离**。说话人验证最终要比较 embedding 余弦相似度，如果训练时只是「分对类」，embedding 空间可能很乱（同类可能离得远，异类可能离得近）。

→ 需要**基于角度的 margin 损失**，强制同类拉近、异类推开。

### 4.2 AAM-Softmax（Additive Angular Margin Softmax）原理

```python
x_norm = F.normalize(embedding, dim=1)      # embedding 归一化到单位球面
w_norm = F.normalize(self.weight, dim=0)     # 每类权重归一化
cos_theta = x_norm @ w_norm                  # 余弦相似度 = cos(θ)
# 在目标类的角度上叠加 margin：
cos_theta_m = cos_theta*cos_m - sin_theta*sin_m   # = cos(θ + m)
output = cos_theta*(1-one_hot) + cos_theta_m*one_hot
loss = cross_entropy(output * scale, labels)
```

**知识点链**：
1. **L2 归一化**：把 embedding 和权重都投影到单位超球面。这样相似度 = 夹角余弦，**与向量长度无关**，只看方向。
2. **margin m=0.2**：在目标类的角度上**额外加一个角度**，要求 `cos(θ_y + m)` 仍要最大才能分对 → 强迫网络把同类压到更小的角度（更紧凑）。
3. **scale s=30**：softmax 对 logit 数值敏感，余弦值范围只有 [-1,1]，差距太小分不开。乘 s 放大差距，让 softmax 有足够梯度。
4. **θ > π-m 的兜底**：当角度已经很大（接近错分类）时，`cos(θ+m)` 会异常，代码用 `cos(θ) - mm` 兜底，保证数值稳定与梯度方向正确。

**为什么 AAM-Softmax 而非 ArcFace**：
- ArcFace：`cos(θ + m)` 在 **arccos 空间**加 margin（要先 arccos 再加再 cos）。
- AAM-Softmax：直接在 **cos 空间**用三角恒等式实现 `cos(θ+m)`，**数值更稳定、实现更简单**。
- 两者效果接近，AAM-Softmax 在工程实现上更不容易出 NaN。本工程两者都提供，默认 AAM。

**我的思考**：margin 损失是声纹模型区别于普通分类模型的「灵魂」。没有它，就算 backbone 再强，embedding 空间也不会自然形成可验证的结构。

### 4.3 AMP（混合精度）兼容处理

代码里把 `cos_m`、`sin_m` 等常量转成与 `cos_theta` 同 dtype 的 tensor：
```python
cos_m = torch.tensor(self.cos_m, dtype=dtype, device=device)
```
**为什么**：半精度（fp16）下，Python float 常量会被提升成 fp32，与 fp16 张量运算会报 dtype 不匹配。这是混合精度训练的常见适配点。

## 5. 训练循环：Trainer

代码：`src/training/trainer.py`

### 5.1 优化器与学习率

```yaml
optimizer: SGD, lr=0.1, momentum=0.9, weight_decay=0.0005
warmup_epochs: 5, lr_schedule: cosine, grad_clip: 1.0
```

#### 为什么用 SGD 而不是 Adam
- **说话人识别社区经验**：SGD+momentum 收敛更稳，最终 EER 通常优于 Adam。
- Adam 收敛快但容易停在次优解；SGD 慢但泛化更好。
- 这是业界共识（ResNet 系列声纹模型普遍 SGD）。

#### Warmup + Cosine 调度
```python
if epoch < warmup_epochs:
    lr = base_lr * (epoch+1) / warmup_epochs        # 线性升温
else:
    progress = (epoch-warmup)/(...)
    lr = base_lr * 0.5*(1+cos(pi*progress))         # 余弦退火
```
- **Warmup**：训练初期权重随机，大 lr 会导致损失爆炸/发散。前 5 epoch 线性升到 0.1，让模型先「站稳」。
- **Cosine 退火**：后期平滑降到接近 0，精细收敛。
- **为什么不用阶梯降**：余弦更平滑，省去调降 lr 时机，现代训练标配。

#### 梯度裁剪 `grad_clip=1.0`
```python
torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
```
- margin 损失早期梯度可能很大，裁剪防梯度爆炸。

### 5.2 训练一个 epoch 的流程

```python
for features, labels in train_loader:
    with autocast("cuda"):              # 混合精度前向
        embedding, logits = model(features)
        loss = criterion(embedding, labels)   # AAM-Softmax
    scaler.scale(loss).backward()       # 放大梯度防 fp16 下溢
    scaler.unscale_(optimizer)
    clip_grad_norm_(...)
    scaler.step(optimizer); scaler.update()
    optimizer.zero_grad()
```

#### 知识点：混合精度训练（AMP）
- 前向用 fp16（快、省显存），反向时 `GradScaler` 把损失放大，避免小梯度在 fp16 下变成 0。
- `scaler.unscale_` 后才能正确裁剪梯度（裁剪要在真实梯度上进行）。
- **本工程实际配置 `amp: false`**（见日志 `AMP: OFF`），即用的全精度训练。可能因为 RTX 5070 Ti 显存够，或为避开 AAM-Softmax 的半精度数值问题。

### 5.3 验证：基于 embedding 的 EER 评估（重点）

`validate_embedding` 不看分类准确率，而是**直接评估 embedding 质量**：

```python
embeddings = extract_embeddings(model, val_loader)   # L2 归一化
# 正样本对（同人）：cos 相似度
pos_scores = [cos(emb_i, emb_j) for 同一说话人 i,j]
# 负样本对（不同人）
neg_scores = [cos(emb_i, emb_j) for 不同说话人 i,j]
eer, _ = compute_eer(pos_scores, neg_scores)
```

#### 知识点 1：余弦相似度打分
验证阶段不分类，而是 `cos(embed_a, embed_b)`，>阈值判为同人。这与训练目标（margin 损失优化角度）**完全对齐**。

#### 知识点 2：EER（等错误率）
```python
for threshold in thresholds:
    far = mean(neg_scores >= threshold)   # 错误接受率
    frr = mean(pos_scores < threshold)    # 错误拒绝率
    eer = (far + frr)/2
```
- **FAR**：不同人被误判为同人。
- **FRR**：同人被误判为不同人。
- 阈值越低 → FAR↑ FRR↓；阈值越高 → FAR↓ FRR↑。
- **EER = FAR == FRR 的交点**，是与阈值无关的单一指标，**越小越好**。
- 这是说话人验证的国际标准指标。VoxCeleb-O 上 SOTA 约 **1~3%**。

#### 知识点 3：minDCF
考虑了误判代价和先验的加权指标，更贴近实际部署。同样越小越好。

### 5.4 Checkpoint 与早停

- 每 epoch 存 `last_model.pth`，EER 最优存 `best_model.pth`，每 5 epoch 存带 epoch 号的快照。
- `early_stopping_patience`：EER 连续 N 个 epoch 不提升就停。
- 实际日志：patience=25，跑到 epoch 71 早停。

## 7. 实际训练日志的诊断（重要，需诚实说明）

阅读 `checkpoints/training.log`，发现几个**必须指出的问题**：

### 7.1 训练准确率≈0.001 —— 指标 bug，非训练失败

日志：
```
Train Loss: 0.0020 | Train Acc: 0.0011
```

`Trainer._train_one_epoch` 里：
```python
correct += (logits.argmax(dim=1) == labels).sum().item()
```
- 用 AAM-Softmax 时，`loss = criterion(embedding, labels)` **只更新 AAMSoftmaxLoss 内部权重矩阵**，根本不碰 `model.classifier`。
- 所以 `logits`（来自 `self.classifier`）几乎是随机初始化值，`argmax` 命中率 ≈ 1/num_speakers ≈ 1/1089 ≈ **0.0009**。
- 这与日志的 `Acc: 0.0011` **完全吻合**。

**结论**：这个 `train_acc` 数字是**无效指标**，不代表模型没在学。真实信号是 **EER**。

**修复建议**：AAM-Softmax 模式下，应基于 AAM-Softmax 的内部 output（带 margin 的 logits）算准确率，或直接只看 EER，删掉这个会误导人的 acc。

### 7.2 EER 先降后升 —— 过拟合 / 模型未充分学到判别性 embedding

两次训练的 EER 曲线：
```
最好 EER=0.1347 → 之后一路升到 0.23 触发早停
另一次最好 EER=0.1553 → 升到 0.249 早停
```

- EER 13~15% 在 VoxCeleb 上**明显偏高**（SOTA 1~3%，一般实现也该进个位数%）。
- 后期 EER **反弹**是典型过拟合：训练集 loss 降到 0.001，但验证集 embedding 质量变差。
- 更关键的信号：`Pos mean`（同人余弦相似度）只有 **0.16~0.36**。健康模型该 >0.7。说明 **embedding 没有把同类拉得足够近**，margin 损失没有真正生效到位。

### 7.3 可能的根因（我的推断）

1. **特征维度不一致**：实际 81 维（含能量）vs 配置写 80，虽不致命但不规范。
2. **AMP 关闭 + batch=128**：AAM-Softmax 在大类别数下，scale=30 + 全精度本应够，但 margin=0.2 偏小（社区常用 0.2~0.35，结合更大 scale）。可尝试 margin=0.2/0.3 + scale=32。
3. **没有数据增强**：`data_config.yaml` 配置了 speed_perturb/noise/spec_augment，但 `SpeakerDataset.__getitem__` 里 `transform=None`（`train.py` 没传 transform）→ **增强全部没生效**。这是过拟合和 EER 偏高的最主要嫌疑。
4. **fixed_length=200 帧（~2秒）** 偏短，说话人信息略不足；`data_config` 里写的 300 没被训练用上。
5. **early_stopping_patience** 配置 25 但 trainer_config 传的是 `early_stopping_patience`，日志显示 patience=25（两次不一致：一次15一次25，说明两次运行配置不同）。

### 7.4 综合判断

- 架构、损失、池化、划分方式 **设计都是对的、符合主流做法**。
- 工程落地有两个硬伤：**(a) 训练 acc 指标在 AAM 模式下失效、(b) 数据增强没接入**。
- 当前 `best_model.pth`（EER≈0.135）**不可直接用于生产**，需修复增强 + 重新训练。

## 结语

这个工程的**架构与算法选型是扎实且符合主流的**：FBank 特征 → ERes2NetV2（分组复用+SE+选择性残差）→ AttentiveStatsPool → AAM-Softmax → 余弦 EER 验证，每一环都有明确的理论依据和社区惯例。

但**工程落地存在两个影响训练效果的硬伤**：数据增强未真正接入、训练准确率指标在 margin 损失下失效。这两点直接解释了日志中「acc≈0」「EER 偏高且反弹」的现象。

修复 P0 两项后重训，EER 有望从 ~13% 进入个位数百分比区间，达到该架构应有的水平。
